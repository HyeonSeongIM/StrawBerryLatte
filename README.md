# StrawBerryLatte


## 요구사항 분석하기
**< 데이터베이스에 대한 사용자들의 요구 사항을 수집하고 분석하여 개발할 데이터베이스의 용도를 명확히 파악하는 것 >**

### 상품 목록
* 상품 목록은 슬라이스 형식으로 보여준다.
* 상품 목록이 보일 때는 상품 사진, 제목, 가격, 좋아요 순도 보여준다.
* 맨 왼쪽에는 카테고리로 정렬할 수 있는 기능이 있다.
* 상품 목록을 표시하기 위해서는 카테고리, 최신순, 오래된순, 좋아요 순으로 나눠져야한다.
* 상품은 카테고리와 생성일자와 좋아요 컬럼이 존재해야한다.

### 상품 상세
* 제목이 맨위에 와야한다.
* 상품의 사진들을 중앙에 배치한다.
* 가격은 사진 아래에다가 둔다.
* 내용은 가격 아래에다가 서술하여 200자 이내로 작성하게 한다.
* 상품은 제목, 사진, 가격, 내용 컽럼이 존재해야한다.

### 리뷰
* 리뷰는 점수와 내용으로 구성된다.
* 점수는 1 ~ 10점까지 정해져 있어 소수점은 존재하지 않는다.
* 내용은 100자 내외로 구성한다.
* 리뷰는 제목, 내용, 점수 컬럼이 존재해야한다.

### Q&A
* 질문은 공개와 비공개로 나뉜다.
* 답변은 한번만 할 수 있다.
* Q&A 테이블을 하나 만들어서 둘이 비교할 수 있는 컬럼을 넣을지
* 아니면 Question과 Answer로 나누어서 저장할지

### 찜
* 찜은 제한이 없다.
* 찜은 좋아요와 같다.
* 사용자의 테이블에 있어야할지.. 상품 테이블에 있어야할지..
* 따로 테이블을 둬야할지..

### 포인트
* 결제 후 0.1% 해당하는 금액을 포인트로 적립시켜준다.

### 쿠폰
* 쿠폰의 유효기간은 각기 다르다.
* 쿠폰의 사용가능 횟수는 1매당 1번이다.

### 장바구니
* 카테고리 별로 장바구니에 있는 상품을 볼 수 있다.

### 주문
* 주문은 상품의 갯수에 맞춰 주문이 가능하다. (애매한 요구사항?)
* 주문은 장바구니 주문과 개별 상품 주문이 가능하다.
* 여러 개의 상품을 장바구니에서 동시에 주문하면 주문 한건 당 하나의 컬럼을 둔다?

### 결제
* 결제는 토스 페이먼츠와 개인 페이먼츠로 가능하다.

### 취소
* 취소는 관리자의 승인이 나기 전까지 가능하다.
* 주문 테이블에서 확정과 취소를 볼 수 있는 애트리뷰트 하나를 생성한다.

### 정산
* 포인트와 결제 가격으로 정산을 진행할 수 있다.
